  ## 作業内容
    コマンド(rndc)でDNSサーバーを制御する

    HOST : vm12
    IPV4 : 192.168.122.12
    MEMO : マスター

    HOST : vm13
    IPV4 : 192.168.122.13
    MEMO : スレーブ
  
  ## コマンド(rndc)
    rndc -b XX  // IPアドレス(XX)を指定
    rndc -c XX  // 設定ファイル(XX)を指定
    rndc -s XX  // ホスト名(XX)を指定
    rndc -p XX  // ポート番号(XX)を指定
    rndc -k XX  // 設定ファイル(XX)を指定
    rndc -y XX  // RNDCキー(XX)を指定
    rndc <上記オプション> reload         // 設定ファイルとゾーン情報をリロード (シリアル番号更新時のみ)
    rndc <上記オプション> reload XX      // ゾーン情報(XX)のみリロード (シリアル番号更新時のみ)
    rndc <上記オプション> reconfig       // 設定ファイル(named.conf)と新規追加のゾーン情報のみリロード
    rndc <上記オプション> stats          // 統計情報をファイル出力
    rndc <上記オプション> status         // ステータス表示
    rndc <上記オプション> dumpdb -all    // ゾーン情報をファイルにダンプ、ゾーンとキャッシュ両方
    rndc <上記オプション> dumpdb -zones  // ゾーン情報をファイルにダンプ、ゾーンのみ
    rndc <上記オプション> dumpdb -cache  // ゾーン情報をファイルにダンプ、キャッシュのみ
    rndc <上記オプション> flush          // 全キャッシュを消去
    rndc <上記オプション> flushname XX   // キャッシュレコード(XX)を消去
    rndc <上記オプション> notify         // マスターからスレーブにゾーン更新を通知 (シリアル番号更新時のみ)
    rndc <上記オプション> retransfer     // スレーブからマスターにゾーン情報の転送依頼 (シリアル番号を無視)
    rndc <上記オプション> refresh        // スレーブからマスターにゾーン情報の転送強制 (シリアル番号を無視)
